# 项目管理登记工具 — 用户使用手册

**手册版本**：1.2（与 **登录账号**、FastAPI 会话鉴权、人员账号管理对齐）

本手册面向 **日常使用者**（含「普通用户」与「管理员」两种 **登录角色**），用通俗语言说明每个菜单是做什么的、该怎么一步步操作。不涉及编程细节。

**页面文件位置**：**`frontend/login.html`**（登录）；主应用在 **`frontend/index.html`**。

---

## 一、开始之前请先了解这几件事

### 1. 怎样打开工具？

推荐用 **Chrome、Edge、Firefox** 等常见浏览器。

- **必须先通过 HTTP 访问**（不要用 `file://` 双击打开），否则 **登录 Cookie 无法正常使用**。请由技术人员用 **网页服务器** 或 Live Server 等，用类似 `http://127.0.0.1:某端口/frontend/login.html` 打开 **登录页**。  
- **登录流程**：在登录页输入 **用户名、密码** → 点 **「登录」** → 成功后进入主页面 **`index.html`**。若未登录直接打开主页面，系统通常会 **跳回登录页**。  
- **后台服务**：登录与权限校验需要 **Python 后台（FastAPI）已启动**；若提示无法连接或超时，请联系技术人员检查后台是否在运行、端口是否正确。

更详细的启动步骤见项目根目录的 **`README.md`**。

### 2. 数据保存在哪里？

- **常见情况（默认）**：您点击各页的 **「保存」** 后，内容会存在 **当前这台电脑、当前这个浏览器的「本地存储」** 里（与早期版本行为类似）。  
- **若单位已部署 Python 后台**：**项目阶段状态、部门项目人力登记、项目风险登记** 这三类，在软件对接完成后，还可能（或改为）保存到 **服务器上的数据库**；是否已对接请以技术人员说明为准。  
- **换电脑、换浏览器、清除浏览器数据**，本地那份可能丢失；**务必养成点「保存」的习惯**，重要数据请按单位规定做备份。  
- **谁能编辑、谁能保存**，由您 **登录的账号角色** 决定（**管理员** 或 **普通用户**），**不是在设置里随便切换身份**。

### 3. 两种身份有什么区别？

| 身份 | 您能做什么 |
|------|------------|
| **普通用户** | 查看各页表格和文字；可打开分析图表、操作指引；**不能**改数字和文字、**不能**保存、**不能**增删项目或风险。 |
| **管理员** | 在普通用户基础上，可以增删改项目与部门、填写并 **保存** 各页数据、维护操作指引等。 |

使用完毕后可在顶部或设置中 **「退出登录」**，特别是在 **共用电脑** 上。权限说明在 **「设置」→「权限管理」**（见本文第八部分）。

### 4. 顶上一排主菜单是什么？

点击即可切换不同页面，从左到右一般为：

1. **项目阶段状态**  
2. **部门项目人力登记**  
3. **项目风险登记**  
4. **PM操作指引**  
5. **设置**

下面按菜单逐一说明。

## 二、项目阶段状态

### 这是做什么的？

按 **某年、某月**，给每个 **子项目** 写当月的工作小结类文字，例如目标、交付、亮点、不足、下月注意点。  
**项目集、项目名称以这一页为准**；人力表里的项目列表会跟这里保持一致（由系统自动对齐）。

### 普通用户怎么用？

1. 点击顶部 **「项目阶段状态」**。  
2. 在上方选好 **年份** 和 **月份**。  
3. 向下滚动表格，阅读每个项目右侧多列文字框里的内容（只读，不能改）。

### 管理员额外可以怎么做？

1. 同上先选好年月。  
2. 在文字框里 **直接输入或修改** 内容。  
3. 需要调整项目结构时：  
   - 点 **「+ 新项目集」** 增加一整组项目；  
   - 某一组右侧有 **「+」** 可 **加子项目**；  
   - 点击项目集或项目名称可 **改名**；  
   - **「×」** 可删除项目集或单个项目（会弹出确认，删除后人力里对应行也会一起变化）。  
4. 改完后务必点页面下方的 **「保存」**，否则关闭浏览器后可能丢失。

### 「项目阶段状态分析」按钮

点开后会弹出窗口，显示当前所选年月；**具体分析内容后续可能增强**（例如自动总结），目前以说明为主。

## 三、部门项目人力登记

### 这是做什么的？

按 **部门列** 登记各项目占用的人力人数（人数为 **0～99** 的整数）。  
上面有 **月度、季度、年度** 三种看法：  
- **月度**：录入和修改数字（仅管理员，且需在「月度」下）。  
- **季度 / 年度**：自动按规则汇总，**只读浏览**。

**重要**：这里的 **项目集、项目名称** 与 **「项目阶段状态」** 里一致；要改名字或增减项目，请在阶段页操作（管理员），人力表会跟着更新。

### 普通用户怎么用？

1. 点击 **「部门项目人力登记」**。  
2. 需要看整年趋势时，点 **「季度」** 或 **「年度」**，再选年份（季度页还要选季度）。  
3. 在 **「月度」** 下可选年月，查看当月各部门、各项目的人数（只读）。  
4. 点 **「项目月度人力分析」**，可在弹窗里看 **饼图、条形图**（当月汇总；若当月全是 0，可能没有图）。

### 管理员额外可以怎么做？（务必在「月度」下操作）

**登记人数**

1. 确认当前在 **「月度」** 子页。  
2. 选好 **年份、月份**。  
3. 在表格里 **点击某个人数格子**，输入数字后 **点击别处或按回车** 结束编辑。  
4. 全部改完后点 **「保存」**。

**改部门表头（一级分组、二级列）**

1. 仍在 **「月度」**。  
2. 表头上可 **改分组名、列名**（按页面提示操作）。  
3. 用 **「+ 部门分组」** 等按钮增加结构（具体以界面按钮为准）。  
4. 改结构后同样要点 **「保存」**。

**项目结构**

- 在人力页也可以点 **「+ 新项目集」** 等，效果与阶段页维护的是 **同一套项目**；更推荐在 **项目阶段状态** 统一管理。

**列宽**

- 月度表部分列宽可以 **拖动调整**，浏览器会记住您的宽度习惯。

## 四、项目风险登记

### 这是做什么的？

做一张 **风险清单**，每条记录包括：关联 **项目**、登记时间、登记人、说明、评估、等级、当前状态、预计解除日期等，方便例会跟踪。

### 普通用户怎么用？

1. 点击 **「项目风险登记」**。  
2. 阅读表格中的每一行。  
3. 需要按某一列排序时，点 **表头里该列的文字按钮**（会显示升序/降序提示），可多试几次切换顺序。  
4. 点 **「项目风险分析」**，查看 **状态分布饼图**、**分组条形图**；条形图旁可切换排序方式（按数量 / 按组内最高等级等）。

### 管理员额外可以怎么做？

1. 点 **「+ 新增风险」** 增加一行。  
2. 在各格中填写或选择内容；**等级** 一般为下拉选项（低、中、高、极高）；**预计解除时间** 可用日期选择。  
3. 不需要的行可点该行的 **「删除」**（以页面为准）。  
4. 修改后点 **「保存」**。

## 五、PM 操作指引

### 这是做什么的？

把常用规范、流程说明放在四个板块里，方便同事 **自助查阅**：

- 产品设计  
- 技术设计  
- 开发 + 测试  
- 运营  

### 普通用户怎么用？

1. 点击 **「PM操作指引」**。  
2. 在四个板块里找到感兴趣的 **标题**，用鼠标 **点击标题**：  
   - 若维护者填了 **网页链接**（以 http 或 https 开头），会在 **新标签页** 打开；  
   - 若没有链接或链接无效，会弹出窗口显示 **文字说明**。  
3. 页面底部可能有联系 PMO 等提示，可按需联系。

### 管理员额外可以怎么做？

1. 进入 **「PM操作指引」**。  
2. 在某个板块下点 **「+ 添加指引」**，填写 **标题**（必填）、**说明正文**、**链接**（选填，需完整网址）。  
3. 保存后列表会更新；已有条目旁一般有 **编辑 / 删除**（以界面为准）。

## 六、设置

### 这是做什么的？

查看 **当前登录账号** 与 **管理员/普通用户** 在各菜单的大致能力说明；**管理员** 还可在此维护 **人员账号**（增删改用户等，具体以界面为准）。

### 使用步骤

1. 点击顶部 **「设置」**。  
2. 左侧点 **「权限管理」**（默认一般是这一项）。  
3. 页面上会显示 **当前用户名** 以及您是 **管理员** 还是 **普通用户**（由登录账号决定，**不能在本页改成另一个人**）。  
4. 下方表格说明：**项目阶段状态、部门项目人力登记、项目风险登记、PM操作指引、设置** 在两种角色下的大致权限。  
5. **管理员** 可见 **「人员账号」** 区域：可添加用户、调整角色等（需后台服务正常）；**普通用户** 会看到说明性提示，无法管理他人账号。  
6. 使用 **「退出登录」** 可结束当前会话并返回登录页（导航栏或设置内入口以界面为准）。  
7. **「通用设置」** 目前为预留入口，以后可能有更多选项。

### 给普通用户的提示

若您发现 **不能点保存、不能编辑**，说明当前登录账号是 **普通用户**。需要录入数据时，请使用 **管理员账号** 登录，或由管理员在 **人员账号** 中为您调整角色（若单位允许）。

## 七、保存习惯建议（尤其管理员）

1. **每个菜单的数据是分开保存的**：在「部门项目人力登记」点了保存，**不会**自动保存「项目风险登记」里的修改。  
2. 改完 **阶段、人力、风险** 后，请到 **对应页面** 点各自的 **「保存」**。  
3. **登录状态** 由服务器会话与浏览器 Cookie 维持；**月度表列宽**、**PM 操作指引** 等仍主要保存在 **本机浏览器**。  
4. 操作指引在弹窗里保存后，一般会 **自动写入本机**（以实际提示为准）。  
5. 保存成功时，页面通常会有 **简短提示**；若提示失败，可能是浏览器禁止存储、空间不足，或 **后台未启动 / 网络不通**。登录时若长时间无反应，也可能是 **API 不可达**：请确认后台已启动后再试。  
6. **首次部署** 时，技术人员可能提供默认管理员账号（如用户名 **Sky**、初始密码以单位配置为准，常见默认见 **`README.md`**）；首次登录后 **请尽快修改密码**（若界面提供修改入口或由管理员在人员账号中重置）。

## 八、常见问题（非技术版）

**问：为什么我改不了数字？**  
答：您可能是 **普通用户** 身份，或人力页不在 **「月度」** 子页；部门结构类操作也必须在月度下。

**问：为什么人力表里的项目和阶段页不一样？**  
答：请以 **项目阶段状态** 为准；刷新或切换页面后，人力表会与阶段表对齐。若仍不对，请管理员检查是否在阶段页保存过。

**问：换了一台电脑还能看到数据吗？**  
答：若数据只存在 **本机浏览器** 里，**默认不能**在另一台电脑直接看到。若单位已部署 **后台服务** 且前端已对接，三类登记表有可能集中在服务器上，换电脑仍可通过同一访问地址查看（需技术人员配置好地址与权限）。具体以单位说明为准。  
若需要带走或备份，请按单位规定导出或联系管理员。

**问：「项目阶段状态分析」里为什么是空的？**  
答：该功能预留用于以后自动分析；当前以占位说明为主。

**问：技术人员说的「要起后端」是什么意思？**  
答：指在本机或服务器上运行项目里的 **Python 后台程序**，用来提供 **登录、权限与数据接口**。一般用户只需按 **`README.md`** 用浏览器打开正确网址并 **先登录**；自己不必操作命令行，除非您负责维护环境。

**问：登录页提示无法连接或超时？**  
答：多为 **后台未启动**、**端口不对** 或 **防火墙/代理** 拦截。请确认地址栏访问的是 **http 开头的页面**（不要 `file://`），并由技术人员确认 API 服务已运行。登录请求有约 **10 秒超时**，超时后请稍候再试或检查网络。

**问：提示用户名或密码错误？**  
答：请确认大小写、空格；若仍失败，联系管理员 **重置密码** 或检查是否使用了正确的环境（测试/正式）。
